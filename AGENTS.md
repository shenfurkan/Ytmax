# AGENTS.md — YTmax

## Project summary
A single-package Python desktop app (CustomTkinter GUI) for downloading YouTube/playlist videos via yt-dlp. No tests, no CI, no linters.

## Commands

```powershell
# Install Python deps
pip install -r requirements.txt

# Run the app (dev)
python main.py

# Build distributable (PyInstaller)
python build.py
#    DEBUG=1 python build.py   → adds a debug onedir build first, then the release onefile
```

## Prerequisites (not pip-installable)

The app **requires** these installed on the system PATH:
- **FFmpeg** — used for video+audio merging and subtitle conversion
- **Node.js** — used by yt-dlp's `js_runtimes` engine; without it, downloads fail with "Requested format is not available"

Missing deps show as red status dots in the status bar. Install via:
```powershell
winget install -e --id Gyan.FFmpeg
winget install -e --id OpenJS.NodeJS
```

## Architecture

```
main.py          — entrypoint, CustomTkinter GUI, all UI logic (~1000 lines)
                    custom PIL-drawn icon system (_mk_icon / _draw_* functions)
                    cubic-ease-out resize animation (_animate_resize)
                    spinner animation (_start_spinner / _stop_spinner)
downloader.py    — core engine: metadata fetcher + download worker using yt-dlp
                    contains VideoInfo, PlaylistInfo, DownloadTask, PlaylistDownloadTask dataclasses
build.py         — PyInstaller wrapper; reads DEBUG env var, produces dist/
YTmax.iss        — Inno Setup installer script (requires dist/YTmax.exe built first)
YTmax.spec       — PyInstaller spec (auto-generated by build.py; gitignored)
YTmax_debug.spec — same, for the debug onedir build
```

The app is a single window (`YtmaxApp(ctk.CTk)`) with collapsible panels: search bar → video/playlist card → settings panel. Cards use cubic-ease-out animation (`_animate_resize`) for smooth expand/collapse.

## Playlist support

- `PlaylistDownloadTask` uses `video_urls` directly (URLs gathered from selected checkboxes) rather than re-fetching the playlist during download.
- `fetch_playlist_info` uses `extract_flat: "in_playlist"` for metadata. Duration/thumbnail may be sparse for large playlists; the UI gracefully handles missing durations.
- `is_playlist_url` matches `youtube.com/playlist?list=…` and any URL containing `list=` parameter.
- Individual video downloads in a playlist are sequential (one at a time), each with its own yt-dlp extraction.

## yt-dlp details

- **Every** yt-dlp call sets `js_runtimes: {"node": {}}` — this is essential to bypass YouTube's bot/client restrictions.
- `extractor_args` forces `player_client` to `["tv", "android", "ios", "web_creator"]` to avoid SABR blocking.
- Downloads run in daemon threads with progress hooks calling back to the UI via `tk.after()`.
- Cookies file path defaults to `~/Downloads/youtube.txt` (Netscape format). The file is only read; never written by the app.

## Build notes

- `dist/` and `build/` are gitignored (PyInstaller output).
- `Version/` (capital V) is **not** gitignored (the gitignore has `version/` lowercase). It contains a pre-built `YTmax.exe` distribution. Do not commit new build artifacts there.
- Inno Setup installer (`YTmax.iss`) expects `dist\YTmax.exe` to exist. Run `python build.py` first.
- PyInstaller hiddenimports: `yt_dlp`, `yt_dlp_ejs`, `customtkinter`, `PIL`, `downloader`.

## Cookies file

- Must be Netscape-format `cookies.txt` exported from a browser extension ("Get cookies.txt LOCALLY").
- Default path: `~/Downloads/youtube.txt`.
- Required for age-restricted videos and to pass bot/sign-in checks.
- Validation logic (`validate_cookies`) checks for Netscape header and the presence of auth cookies (SAPISID, SID, HSID, SSID, LOGIN_INFO, etc.).

## Windows-specific

- The app uses `sys._MEIPASS` for PyInstaller resource paths, falling back to `os.path.abspath(".")` during dev.
- Window icon: `.ico` on Windows, `.png` on other platforms.
- The status bar shows Node.js and FFmpeg availability with colored dots (green=found, red=missing).
